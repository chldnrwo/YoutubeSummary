# 📝 업데이트 노트 (2026-04-13)

## 📌 주요 업데이트: 오라클 클라우드 성공적 배포 및 보안 강화

오늘 수행된 작업을 통해 서비스가 로컬 환경을 넘어 공식적인 클라우드 도메인 환경으로 이전되었으며, 보안 수준이 대폭 향상되었습니다.

### 1. 서비스 인프라 구축
- **고정 도메인 연결**: `insightpipeline.duckdns.org` 주소를 통해 서비스 접근이 가능합니다.
- **HTTPS 보안 적용**: Nginx와 Let's Encrypt(Certbot)를 연동하여 SSL/TLS 보안 인증서를 배포했습니다.
- **포워딩 설정**: 80(HTTP) 요청을 자동으로 443(HTTPS)으로 리디렉션하며, 내부적으로는 8501(Streamlit)로 안전하게 연결합니다.

### 2. 구글 OAuth 로그인 장애 해결
- **PKCE 표준 도입**: 브라우저와 서버 간의 안전한 토큰 교환을 위해 `code_verifier`를 사용하는 PKCE 방식을 전면 적용했습니다.
- **인증 상태 영구 보관**: 스트림릿의 세션 유실 문제를 해결하기 위해, OAuth 인증 상태(`state`)를 SQLite DB(`oauth_states` 테이블)에 임시 보관 후 콜백 시 복구하는 이중 안전 시스템을 구축했습니다.

### 3. 긴급 보안 세척 (Git History Cleanup)
- **민감 데이터 제거**: 실수로 깃허브 공개 리포지토리에 유출된 `insights.db` 및 `config.json` 파일을 모든 과거 커밋 기록에서 완전히 삭제했습니다.
- **보호 강화**: `.gitignore` 파일을 재정비하여 향후 데이터 유출 사고를 방지하도록 조치했습니다.

### 4. 기타 개선 사항
- **HTTPS 자동 감지**: 앱이 실행 환경(도메인 여부)을 스스로 판단하여 `http` 또는 `https` 프로토콜을 유연하게 선택하도록 로직을 보강했습니다.
- **다중 계정 로그인 대응**: 크롬 등 브라우저의 다중 계정 세션 충돌 시 대처 방안(시크릿 모드 사용 권장)을 가이드에 추가했습니다.

---
**💡 관리자 참고**: 
서버의 Git 기록이 재작성(rewrite)되었으므로, 서버에서 업데이트 시 반드시 `git fetch` 후 `git reset --hard origin/main`을 사용하여 동기화해야 합니다.
