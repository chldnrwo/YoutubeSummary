# 📋 Insight Pipeline 개발 경과 보고서 (Update Note)

이 문서는 최근 진행된 **'UI 레이아웃 안정화(스크롤 버그 해결)'** 및 **'유튜브 실제 업로드 시간(published_at) 트래킹 구조'**에 대한 변경 사항과 워크플로우를 정리한 문서입니다.

## 🎯 주요 이슈 및 해결 방안

### 1. 극심한 화면 밀림 (Layout Shift) 현상 및 스크롤 버그
* **문제 상황:** 구독 피드 화면에서 [숨기기] 버튼이나 [분석] 버튼을 짧게 연속으로 클릭할 시, 화면이 상하로 심하게 요동치며 스크롤이 맨 밑으로 처박히는 버그가 존재했습니다.
* **해결 과정 핵심 아이디어:** Streamlit의 렌더링 특성상, DOM 구조(버튼, 텍스트 높이, CSS 공백)가 미세하게 1px이라도 달라지면 브라우저가 화면을 억지로 다시 그리면서 스크롤 위치를 잃어버리게 됩니다.

**[작업 내역]**
* **버튼 레이아웃 100% 동일화:** `분석 전`, `분석 중(info)`, `분석 완료(success)`, `에러(error)` 등 모든 처리 상태의 알림 메시지를 삭제하고, 기본 상태와 완전히 똑같은 구조의 `st.columns([3, 1])` 및 비활성화(Disabled)된 버튼 컴포넌트로 일괄 교체했습니다.
* **숨김 처리(Hide) 카드 Parity:** 숨김 처리된 카드는 즉시 사라지지 않고, 기존 썸네일과 동일한 16:9 뼈대를 유지한 채 흑백(Grayscale) 필터와 취소선만 적용하여 화면의 그리드가 무너지는 빈자리를 방지했습니다. (새로고침 시에만 리스트에서 날아감)
* **보이지 않는 CSS 공백 제거:** `<style>` 태그 렌더링 시 발생하는 유령 DOM(보이지 않는 마진 요소)을 `video-title` div 안으로 병합하여 CSS 마진 버그를 완벽 처단했습니다.

## 📅 데이터 연동 및 DB 스키마 구조 변경

### 2. 유튜브 실제 업로드 시간 트래킹
* **문제 상황:** SQLite의 `insights` 테이블에는 유저가 분석 버튼을 눌러 데이터를 '저장한 시점(`created_at`)'만 기록될 뿐, 며칠 전에 유튜브 채널에 영상이 최초로 올라왔는지에 대한 메타데이터(`published_at`)가 소실되는 문제가 있었습니다.

**[작업 내역]**
* **Migration:** `init_database()`에 `ALTER TABLE` 구문을 추가하여, `insights` 메인 테이블에 `published_at TIMESTAMP` 컬럼을 신규 확장했습니다.
* **데이터 파이프라인 (Data Pipeline):**
  1. `get_subscriptions()`: 유튜브 Data API를 통해 구독 플레이리스트 조회 시, `publishedAt` 변수를 배열에 함께 파싱하여 메모리에 적재합니다.
  2. `submit_analysis()` -> `analyze_video()`: 백그라운드 ThreadPool 구동 시 위에서 캐치한 업로드 날짜를 인자로 넘겨 같이 들고 뛰게 만들었습니다.
  3. `save_insight()`: 제미나이 분석 종료 직후, 트랜잭션 SQL 인서트 수행 시 `published_at` 값을 함께 SQLite에 Write합니다.

**[UI 동작 로직 분기]**
* **구독 피드 기반 분석:** 이미 유튜브 API에서 수집된 업로드 정보(`published_at`)가 정상적으로 DB에 기록되며, 사이드바와 메인 뷰에 달력 아이콘(`📅 YYYY-MM-DD`)과 함께 노출됩니다.
* **URL 클립보드 기반 분석:** 유튜브 데이터를 가져오지 않는 모드이므로 API 호출이 생략되어 `NULL`로 처리되며, 기존과 동일하게 플로피 아이콘(`💾 저장일자`)으로 안전하게 폴백(Fallback) 됩니다. (UI 크래시 방지를 위해 `IndexError` 회피 처리 완료)
